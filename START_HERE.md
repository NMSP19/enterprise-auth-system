## The Project in 30 Seconds

Users login with username/password. Backend validates them, creates a JWT token, sends it to frontend. Frontend stores it and includes it in every request. Backend validates token and checks if user has permission. If yes, returns data; if no, returns 403 Forbidden.

**Key Features**:
- ✅ Two user types: Admin (full access), User (limited access)
- ✅ JWT tokens (15-min access, 7-day refresh)
- ✅ Session timeout (30 min inactivity)
- ✅ Role-based access control
- ✅ Production-grade security
